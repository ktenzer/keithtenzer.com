Keith,

First if all kudos for sharing all these information and informative blog posts.
I have been reading them for the last few weeks,especially the posts that reference ovirt/RHEV.
We are testing different virtualisation managers trying to find what is better for our specific problem and i would like your input on it.

We will create a number of small (5-10 servers max) "remote datacenter" that are connected to the "internet" over sat link (so bandwidth is a big issue).

We tried OpenStack but soon realised it was an overkill for the size of the datacenters we have and the amount of work required to set it up and manage/maintain it after.
We briefly looked at cloudstack, but given its state we are not sure if its the correct option for a long-running project.
Recently we have been playing around with RHEV-H/RHEV-M in a local setup to get some first hand experience.

What we are not sure if RHEV-M would operate in the limited bandwidth environment we have when it has to handle more than 50 RHEV-H "datacenters" or we have to deploy one RHEV-M in every remote location to handle the local servers and then write our own management code using the SDK/API to manage the RHEV-M instances.

Given your experience i would appreciate your input on how we could architect a system like that.

Thank you again for all the hard work and info you put on the blog

Best Regards

Stelios Koroneos
